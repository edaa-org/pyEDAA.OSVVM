pyEDAA.OSVVM.Procedures

This module implements OSVVM’s TCL procedures (used in OSVVM’s *.pro files) as Python functions.

These functions are then registered at the TCL processor, so procedure calls within TCL code get “redirected” to these Python functions. Each function has access to a global variable osvvmContext to preserve its state or modify the context.

pyEDAA.OSVVM.Procedures.build(file, *options)[source]

This function implements the behavior of OSVVM’s build procedure.

The current directory of the currently active context is preserved while the referenced *.pro file is processed. After processing that file, the context’s current directory is restored.

The referenced file gets appended to a list of included files maintained by the context.

pro-file discovery algorithm:

  1. If the path explicitly references a *.pro file, this file is used.

  2. If the path references a directory, it checks implicitly for a build.pro file, otherwise

  3. it checks implicitly for a <path>.pro file, named like the directories name.

Parameters:

  • file (str) – Explicit path to a *.pro file or a directory containing an implicitly searched *.pro file.

  • options (int) – Optional list of option IDs.

Raises:

  • TypeError – If parameter proFileOrBuildDirectory is not a Path.

  • OSVVMException – If parameter proFileOrBuildDirectory is an absolute path.

  • OSVVMException – If parameter proFileOrBuildDirectory is not a *.pro file or build directory.

  • OSVVMException – If a TclError was caught while processing a *.pro file.

Return type:

None

See also

  • BuildName()

  • include()

  • ChangeWorkingDirectory()

pyEDAA.OSVVM.Procedures.library(libraryName, libraryPath=None)[source]

This function implements the behavior of OSVVM’s library procedure.

It sets the currently active VHDL library to the specified VHDL library. If no VHDL library with that name exist, a new VHDL library is created and set as active VHDL library.

Hint

All following analyze calls will use this library as the VHDL file’s VHDL library.

Caution

Parameter libraryPath is not yet implemented.

Parameters:

  • libraryName (str) – Name of the VHDL library.

  • libraryPath (Optional[str]) – Optional: Path where to create that VHDL library.

Return type:

None

See also

  • LinkLibrary()

  • LinkLibraryDirectory()
