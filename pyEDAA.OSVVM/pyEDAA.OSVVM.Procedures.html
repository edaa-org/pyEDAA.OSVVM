pyEDAA.OSVVM.Procedures.build(file, *options)[source]ÔÉÅ

This function implements the behavior of OSVVM‚Äôs build procedure.

The current directory of the currently active context is preserved while the referenced *.pro file is processed. After processing that file, the context‚Äôs current directory is restored.

The referenced file gets appended to a list of included files maintained by the context.

pro-file discovery algorithm:

  1. If the path explicitly references a *.pro file, this file is used.

  2. If the path references a directory, it checks implicitly for a build.pro file, otherwise

  3. it checks implicitly for a <path>.pro file, named like the directories name.

Parameters:

  • file (str) ‚Äì Explicit path to a *.pro file or a directory containing an implicitly searched *.pro file.

  • options (int) ‚Äì Optional list of option IDs.

Raises:

  • TypeError ‚Äì If parameter proFileOrBuildDirectory is not a Path.

  • OSVVMException ‚Äì If parameter proFileOrBuildDirectory is an absolute path.

  • OSVVMException ‚Äì If parameter proFileOrBuildDirectory is not a *.pro file or build directory.

  • OSVVMException ‚Äì If a TclError was caught while processing a *.pro file.

Return type:

None

pyEDAA.OSVVM.Procedures.library(libraryName, libraryPath=None)[source]ÔÉÅ

This function implements the behavior of OSVVM‚Äôs library procedure.

It sets the currently active VHDL library to the specified VHDL library. If no VHDL library with that name exist, a new VHDL library is created and set as active VHDL library.

Hint

All following analyze calls will use this library as the VHDL file‚Äôs VHDL library.

Caution

Parameter libraryPath is not yet implemented.
